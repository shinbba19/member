📄 PRODUCT REQUIREMENT DOCUMENT
Project: Kinik Thai Loyalty Point System

Version: 1.0
Architecture: Supabase + Streamlit
Deployment: Streamlit Cloud
Integration: LINE OA (Link Only – No Webhook)

1. Project Objective

Build an internal loyalty system for Kinik Thai Co., Ltd. that:

Awards points based on purchase invoices

Implements FIFO point deduction

Enforces 1-year expiration for both points and coupons

Allows 1,000 points → 50 THB coupon redemption

Uses Supabase as database and business logic layer

Uses Streamlit as frontend (Admin + Customer Portal)

Uses LINE Official Account only as a link to the customer portal

Does NOT use blockchain

Does NOT use n8n

Does NOT require webhook

2. Business Rules
2.1 Point Earning

10 THB = 1 point

Points earned = floor(invoice_amount / 10)

Points are created per invoice as a separate FIFO lot

2.2 Point Expiration

Points expire 1 year after invoice_date

Expiration applies per lot

Only remaining points expire

Expired points must not affect redeemed points

Expired lots must change status to EXPIRED

2.3 Redemption

1,000 points = 50 THB discount coupon

FIFO deduction required

Deduct from:

earliest expires_at

then earliest earned_at

Coupon expires 1 year after creation

Coupon can only be used once

Coupon status must change to USED after redemption

5. Required SQL Functions

Claude must implement:

5.1 create_lot_for_invoice(invoice_no)

Fetch invoice

Calculate points

Insert new lot

Set expires_at = invoice_date + 1 year

5.2 redeem_fifo(customer_id)

Call expire_lots()

Check available balance

If balance < 1000 → error

Create coupon

Create redeem record

Deduct points from lots using FIFO

Insert redeem_allocations

Return coupon code

5.3 expire_lots()

Find lots where:

expires_at <= now()

status = ACTIVE

points_remaining > 0

Set:

status = EXPIRED

points_remaining = 0
